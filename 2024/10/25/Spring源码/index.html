Spring架构概述

整体架构概述

Spring核心概述：

  1. Spring中有一个 ResourceLoader 组件，它负责将我们bean配置文件读取进来。
  2. 使用 Resource 来接收，万物皆可是资源，所以Spring中定义了很多的资源读取方式，常用的是 xml 和 注解的读取方式，当然我们也可以自定义资源读取方式。
  3. 将读取的资源交给对应的 BeanDefinitionRead 来将进行处理然后放入到 BeanDefinitionRegistry 中
  4. BeanDefinitionRegistry 中存储的是一个个bean的定义，叫做 BeanDefinition，遍历获取每一个 BeanDefinition 来进行对象的创建
  5. 最后交给专门创建的对象的来进行创建，将创建好的对象分别放入对应的对象池，其中单例对象放入单例对象池中，需要用到的时候再从中获取出来

Resource

资源抽象接口，将不同的配置对象的方式抽象成接口，Resource 下有很多不同的接口和读取方式，比如使用xml文件来配置对象、使用注解来进行配置、读取网络上的文件来进行配置等等

ResourceLoader

核心方法只有一个

将资源资源文件进行导入变成 Resource
